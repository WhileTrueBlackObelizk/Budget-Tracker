# Contributing to Budget Tracker

Thanks for your interest in contributing! Here's how to get started.

## Getting Started

1. **Fork** this repository
2. **Clone** your fork: `git clone https://github.com/<your-username>/budget-tracker.git`
3. Create a **feature branch**: `git checkout -b feature/your-feature-name`
4. Make your changes
5. **Test** that the app runs correctly
6. **Commit** with a clear message: `git commit -m "Add: short description"`
7. **Push** to your fork: `git push origin feature/your-feature-name`
8. Open a **Pull Request**

## Code Style

- **Python**: Follow [PEP 8](https://peps.python.org/pep-0008/). Use type hints where possible.
- **JavaScript**: Use `const`/`let` (no `var`), template literals, and `async/await`.
- **CSS**: Use the existing custom property system (`--clr-*`, `--radius`, etc.).

## Commit Messages

Use clear, descriptive commit messages:

```
Add: new feature description
Fix: bug description
Refactor: what was improved
Docs: what was documented
```

## Reporting Issues

If you find a bug or have a feature request, please [open an issue](../../issues) with:

- A clear title
- Steps to reproduce (for bugs)
- Expected vs. actual behaviour
- Your environment (OS, Python version)

## Questions?

Feel free to open an issue for any questions about the codebase.
